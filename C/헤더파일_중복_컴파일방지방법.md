# C or C++ 헤더파일 중복 컴파일 방지하는 방법

## 헤더파일이 하는 일
```헤더 파일에 함수 원형들을 공개하고, 다른 소스코드상에서 이 헤더를 포함함으로써 어떤 함수들이 존재하는지 알 수 있다.```

### 중복 컴파일 예시
> #### 중복 컴파일이란 이미 헤더코드를 실행했는데 또 다시 실행되는 것(?)을 말한다. 예시를 들자면 먼저<br> - sub.c : 함수들을 실행하는 기능을 보유 <br> - add.c : 더하는 기능을 보유<br> - show.c : 사용자에게 결과를 보여주는 기능을 보유<br> - show.h : show 헤더파일<br>이렇게 4개의 파일이 있다고 치자.<br>여기서 add.c 에서 show.h 를 include 하고 sub.c 에서도 show.h를 include 하면 이미 정의한 것을 또 한번 더 정의 하였기 때문에 에러가 걸림.<br>이럴 때 `#ifnedf, #define`을 사용하여 해결할 수 있다.                           
> ```
> #ifndef __SHOW__      // 이거 원래 언더바 두개 붙이는게 국롤임
> #define __SHOW__
> #include "show.h"
> 실행할 코드들~~~~~
> ~~~~~~~~~~~~~~~~~
> ~~~~~~~~~~~~~~~~~
> #endif
> ```
> #### __SHOW__가 있나 없나 검사를 함 -> 1. 있다 그러면 #endif 까지 코드를 무효화 시킴  , 2. 없다. 없을 경우에는 실행시킴 아주 간단( define __SHOW__ 해서 정의하니까 한번 실행하면 다음번엔 실행이 안됨
